# 数据纪律

> 这份纪律的目的不是事先规训，而是 v1 失败的总结：v1 跑比亚迪首例时把海豚君的数字直接列表 + 借用对方公式 + 排版成自有计算，结果像 plagiarism。下面三条规矩就是为了避免这类问题再发生。

## 1. 数字标签约定

deep-dive 章节里每个数字应在数字后括注来源：

- 来自外部研报：`(海豚君 2026-04-29 / verified: partial)`
- 来自财报原文：`(2025 年报 p.X / verified: true)`
- 来自公司公告：`(公司销量公告 2026-04-XX / verified: true)`
- 来自 WebSearch 但未回访核对：`(xxx / verified: unverified)` 或 `⚠️待核实`
- 自己用业务数据派生计算：`(自算：销量 70.04 万 × 单价 16 万)`

**为什么要标签**：直接列表格而不标来源，读者会误以为是 wiki 自有结论。比亚迪首例的"单车核心运营利润 0.59 万 / 0.56 万 / +5%"全是海豚君原文数字，但表格无标签，给读者错觉是 wiki 自己算的。这种"借用对方算法 + 借用对方数字 + 排版成自己的"在投研里至少要写来源标签。

引用尚未 WebFetch 原 URL 核对的数据时，标 `verified: partial` 或 `unverified`。partial 是诚实的——读者可以判断该数据可信度。

## 2. 强叙事词处理

腰斩 / 暴跌 / 翻倍 / 最 / 80%+ / 完全 / 永久 / 颠覆 等强词出现时，每处需在该数字后加：

- 已 WebFetch 原 URL 核对 → `(verified: true，来源 xxx)`
- 未核对 → `⚠️待核实` 标签

单项变动 ≥ 50% 或打破历史极值的断言，需给出原始数据源链接（不是汇总文章链接）。

**为什么严格**：CLAUDE.md "叙事去强化"硬约束：看到强叙事词时默认标"待核实"，直到能给出单一权威出处（基金季报 PDF / 官媒原文 / 公司公告）。仅凭多篇二手媒体相互引用不构成可信来源。

## 3. SKU 数据降级规则

不是所有公司都有比亚迪那种月度分车型销量公告。当 SKU 级数据拉不到时，按以下顺序降级——**避免用"安全归纳"敷衍**（如"三款车型表现良好"），那是 v1 失败的伪装：

| 数据可得性 | 操作 |
|---|---|
| 公司有月度分车型销量公告（比亚迪/理想/小鹏）| ✅ 拉到具体 SKU 级，按 Step 2 的 5 项展开 |
| 仅披露品牌/产品线级（长城：哈弗/魏牌/坦克/欧拉/皮卡）| 降级到品牌级 + 找该品牌内主销车型（如长城坦克 → 坦克 300/400/500 单独搜，至少给出主销 1-2 款的具体定价/竞品对位）|
| 仅披露汇总销量（部分港股/老牌车企）| 降级到价格带分析：按 5 万元一档（< 10 / 10-15 / 15-20 / 20-25 / 25-30 / 30+）拆销量分布；并至少为最重要价格带挑 1 款标杆产品做"是什么 / 对位谁"分析 |
| 三方榜单也搜不到 | 明示"分 SKU 数据不可得，仅能做品牌/价格带分析"，不要假装有 SKU 洞察 |

**关键原则**：宁可承认数据稀疏不写，也不要用归纳泛化（"产品矩阵丰富"、"销售表现稳健"）—— 后者是模板填空的伪装，恰恰是 v1 失败的原因。承认数据稀疏对决策有用（说明该公司透明度不足，应作为风险因素），用归纳泛化对决策无用。
