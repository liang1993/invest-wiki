# 价值投资分析 Skill

面向左侧价值投资者的个股分析工具。核心输出：**基于最新股价的投资价值判断和安全边际买入价**。

## When to Use

- 用户要求分析某只股票的投资价值
- 用户询问某只股票是否值得买入
- 用户要求评估持仓股的当前位置
- 用户要求更新 wiki 中个股的估值数据

## 投资者画像

- 价值投资为主，也关注成长
- 持仓周期月起步，偏长期，左侧风格
- 偏向资产配置，很少投机
- 非常看重安全边际，不做止损
- 市场：A股、港股

### 数据获取脚本

`scripts/fetch_data.py` 封装了 AKShare + yfinance 的数据获取逻辑：

```bash
# 用法：python3 scripts/fetch_data.py <A股6位代码> [同行代码1,同行代码2]
python3 scripts/fetch_data.py 603259 300759,002821,300347
```

输出：财务摘要（5年年报+最新季报）、行情数据（价格/市值/52周高低/分红）、单季度扣非趋势、同行对比。

### 数据源说明
- **AKShare**：`stock_financial_abstract(symbol)` 获取财务摘要，稳定可用
- **yfinance**：行情、估值、分红数据。A股代码格式 `600519.SS`（沪）/ `000858.SZ`（深），港股 `0700.HK`
- AKShare 行情接口可能因代理被拦截，行情统一用 yfinance
- 港股财务数据 AKShare 支持有限，优先用 yfinance 的 `ticker.financials`
- 接口报错时说明并跳过，用已获取的数据继续分析

## 分析流程

### Step 0: 估值方法选择（必做，不可跳过）

**目的**：避免"先用错估值方法跑完一遍再发现要返工"的浪费（教训：长城汽车 v1→v2→v3 三轮重写）。

**操作**（按以下顺序，**默认走快速通道，决策树仅作兜底**）：

1. **🚀 优先查快速通道**（[框架.md 默认入口](../../wiki/strategies/估值方法选择框架.md#-默认入口快速通道已分析过的同行业直接套用)）：
   - 分析对象的子行业是否在快速通道表中（汽车/啤酒/白酒/水泥/银行/券商/互联网/晶圆代工/光刻机/有色等 13 个已覆盖）？
   - **命中 + 业务模式相似** → 直接套用主估值，跳到第 4 步输出
   - **命中但业务模式差异大**（如长城出海占比 38% vs 长安 22%）→ 套用方法但调整参数，跳到第 4 步
   - **强制 review 触发**：参照标的 wiki 更新 >6 个月 / 重大事件 / 主业占比 <70% → 不能直接套用，进入第 2 步

2. **🌳 未命中走决策树**（仅新子行业用，[框架.md 兜底入口](../../wiki/strategies/估值方法选择框架.md#-兜底入口决策树新子行业用)）：
   - 拉取近 5-8 年营收 / 销量 / 净利率历史（akshare `stock_financial_abstract` + 必要时 WebSearch 补充销量数据）
   - 按 5 问决策树判断：
     - Q1：行业是否金融？→ 是 → PB 主
     - Q2：营收是否单边上行（5-10 年仅 1-2 年微跌 <5%）？是 → 跳 Q4（不问 Q3）；否 → 进 Q3
     - Q3：销量/产量是否同步周期波动？→ 是 → 真周期股 PB 主
     - Q4：净利率是否长期稳态（5 年波动 ≤±20%，**剔除单年极端值**）？→ 是 → 稳态消费/成长 PE 主
     - Q5：以上都否（营收单边 + 净利率周期 ±50%+）→ 弱周期成长 P/S 主
   - 阈值口径：净利率波动 = `(max - min) / 中位数`（先剔除 >2σ 的单年极端值）

3. **特殊规则**：主业占比 <70% → 必须 SOTP 分部估值（不再走单一主估值）

4. **输出主估值 + 1-2 辅助方法**

**输出格式**（必须显式记录到 wiki 的"企业类型与估值模型"章节）：

```
企业类型：[稳态消费/金融/真周期/弱周期成长/高成长]
主估值：[PE / PB / P/S / PEG]
辅助：[DCF / SOTP / DDM / 中周期 PE]
不适用：[列出明确不能用的方法及原因]
判断依据：[关键 1-2 个数据点，如"营收 8 年 +125% 仅 2019 -3% / 净利率 2-6.3% 区间波动"]
```

**强制规则**：
- **不通过本步骤不进入 Step 1** —— 否则 Step 3 的合理 PE 推导可能完全是浪费
- 如选 P/S 主，跳过 Step 3（合理 PE 因子推导），直接到 Step 4 P/S 模型
- 如选 PB 主（周期/金融），Step 3 简化为基础 PB 推导
- 如选 PE 主（稳态），按 Step 3 完整推导
- **强制引用 [估值方法选择框架](../../wiki/strategies/估值方法选择框架.md) 的子行业 P/S 基准**（避免 skill 与 wiki 数据脱节）；wiki 没列的子行业必须自行补充同行可比表

**反向健康检查**（Step 0 完成后做）：

完成 Step 1-7 跑出合理价值后，**强制回到 Step 0 做以下校验**——若任一项触发，必须复查方法选择是否正确：

| 触发条件 | 含义 | 行动 |
|---------|------|------|
| 合理价值 vs 当前市价偏离 > **±40%** | 极有可能方法选错或参数失真 | 重新走 Step 0 决策树 + 复查同行可比 |
| 合理价值 vs 历史 1 年区间外 | 估值锚脱离市场实际定价 | 检查是否用了周期低位 EPS / 主题泡沫期数据 |
| 三模型（P/S/PE/SOTP）差距 > 30% | 主辅模型分歧 | 优先信营收锚（P/S），复查净利率假设 |
| 与"快速通道"参照标的差距 > 50% | 子行业内异常值 | 检查业务模式是否真的可比 |

**校验示例**（长城汽车 v1 的反例）：
- v1 用 TTM PE 跑出合理价 14 元 vs 当前 20.23 元 = **-30% 偏离**（接近 -40% 阈值）
- 若加了反向健康检查，v1 会被自动拦截，不会等到 v3 才发现问题

### Step 1: 获取数据

同时获取以下数据：

1. **财务摘要**（AKShare）：近5年年报 + 最新季度的关键指标
   - 营业总收入、归母净利润、经营现金流量净额
   - 营业总收入增长率、归属母公司净利润增长率
   - 毛利率、销售净利率、净资产收益率(ROE)、总资产报酬率(ROA)
   - 基本每股收益、每股净资产、每股现金流
   - 资产负债率、流动比率
   - 存货周转天数

2. **行情数据**（yfinance）：
   - 当前股价、市值
   - 52周高低、5年高低
   - 股息率、分红历史

3. **TTM EPS 自行计算**（不使用 yfinance 的 trailingEps）：
   - 公式：TTM净利润 = 最新累计 + 上年未覆盖季度（如：前三季累计 + 上年Q4）
   - 1-4月用上年全年EPS；5-8月用最新累计+上年后续季度补齐；9-12月用前三季+上年Q4
   - 注意季节性（啤酒Q4亏损、白酒Q4最强），TTM自动平滑

4. **每股净资产（用于PB计算）**：优先使用最新季报的每股净资产，不用年报数据。券商等金融股净资产变动较快，用滞后数据会导致PB失真。

5. **同行数据**（AKShare）：同行业2-3家核心公司的财务摘要，用于横向对比

### Step 1B: 利润表分解与EPS基线

**目的**：AKShare 的 `stock_financial_abstract` 只提供汇总指标（毛利率、净利率），无法看到利润表的每一行。对于存在大额非经常性损益、持续/终止经营分拆、SBC费用等复杂情况的公司，仅看汇总指标会导致EPS基线失真。本步骤通过读取年报利润表逐行拆解，建立真实的经营利润基线。

**适用条件**（满足任一即执行）：
- 归母净利润 与 扣非净利润 偏差 > 15%
- 存在终止经营/剥离业务
- 投资收益 / 公允价值变动 占净利润比例 > 10%
- AKShare 的毛利率与 (1 - 营业成本/营业总收入) 计算值不一致
- 公司有大额SBC（科技/CXO/创新药常见）

**不适用**：归母与扣非接近（偏差<10%）、利润结构简单的成熟企业（如茅台、海天）。

#### 1B.2 利润瀑布图

将利润表组织为层次化瀑布，标注每层占收入的百分比和同比变化：

```
营业收入              xxx亿   100%    +xx%
- COGS               (xxx)   -xx%    +xx%
= 毛利                xxx亿   xx.x%   +xx%    ← 真实毛利率（不依赖AKShare）
- 期间费用+税金        (xxx)   -xx%    ← 逐项列示
= 经营毛利             xxx亿   xx.x%   ← 核心经营能力
+ 其他/投资/FV等       xxx            ← 区分经常性 vs 一次性
= 营业利润             xxx亿
- 税/少数股东          (xxx)
= 归母净利润           xxx亿
```

#### 1B.3 非经常性损益桥接

逐项分解从归母到扣非的路径：

```
归母净利润           xxx亿
减：非经常性损益      -xxx亿
  ├ {项目1}          +/-xxx   ← 逐项列出，标注是否来自年报非经常明细表
  ├ {项目2}          +/-xxx
  ├ ...
  ├ 所得税影响        -xxx
  └ 少数股东影响      -xxx
= 扣非归母净利润     xxx亿    ← 用于PE估值的EPS基准

加：SBC add-back     +xxx亿   ← 从"扣除股份支付影响后的净利润"反推
加：其他调整          +xxx亿
= 经调整Non-IFRS     xxx亿    ← 公司自定义口径（如有）
```

#### 1B.4 核心经营利润验证

反推验证扣非净利润与利润表的一致性：

```
核心营业利润 = 营业利润 - 非经常性损益(税前合计)
核心利润总额 = 核心营业利润 ± 营业外
核心所得税 = 总所得税 - 非经常性税影响
核心净利润 = 核心利润总额 - 核心所得税
核心归母 = 核心净利润 - (少数股东 - 非经常少数股东影响)

验证：核心归母 ≈ 扣非归母（偏差应<1%）
→ 输出核心有效税率 = 核心所得税 / 核心利润总额
```

如果验证不通过（偏差>2%），检查是否有遗漏的非经常性项目。

#### 1B.5 利润驱动因子排序

对比本期与上期，按对扣非利润的贡献绝对值排序，列出前5大驱动因子：

```
| 排名 | 驱动因子 | 对扣非利润的贡献(亿) | 说明 |
|------|--------|-------------------|------|
| 1 | {如：毛利率提升} | +xx | {具体数据} |
| 2 | {如：收入增长} | +xx | |
| ... | | | |
```

这一步的输出直接决定了远期分析中哪些变量最需要重点预测。

#### 1B.6 EPS基线决策

根据分解结果，决定PE估值使用哪个EPS：

| 条件 | 使用的EPS | 理由 |
|------|---------|------|
| 归母与扣非偏差<10% | 归母EPS | 差异不大，用报表口径 |
| 偏差10-30%，非经常主要为正向 | 扣非EPS | 非经常收益不可持续 |
| 偏差10-30%，非经常主要为负向 | 归母EPS | 非经常损失已过去 |
| 偏差>30% | 扣非EPS | 必须剔除大额一次性 |
| 存在持续/终止经营分拆 | 持续经营口径的扣非EPS | 终止业务已剥离 |

输出：选定的EPS值及理由，作为后续 Step 3（PE模型）和 Step 4B（远期分析）的输入。

### Step 2: 财务质量评估

对以下维度逐项评分，每项给出具体数据支撑：

**盈利能力：**
- 毛利率水平及5年稳定性
- 净利率水平及趋势
- ROE水平及趋势（杜邦分解：利润率 × 周转率 × 杠杆）

**成长性：**
- 近3年营收复合增速
- 近3年净利润复合增速
- 增速趋势（加速/稳定/减速）

**现金流质量：**
- 经营现金流 / 净利润（>1为优）
- 自由现金流是否充裕

**财务安全：**
- 资产负债率（<40%为优，<60%为良）
- 流动比率（>2为优，>1.5为良）
- 有息负债占比

**分红能力：**
- 股息率
- 分红率及趋势
- 连续分红年数

### Step 3: 合理PE定价模型

> ⚠️ **触发条件**：仅当 Step 0 判定主估值 = PE 时执行本节。
> - P/S 主估值的公司（弱周期成长，如汽车/家电）→ 跳到 Step 4 的 P/S 模型
> - PB 主估值的公司（真周期/金融）→ 跳到 Step 4 的 PB 模型
> - PEG 主估值的公司（高成长）→ 跳到 Step 4 的 PEG 模型


#### 3.1 行业基础PE

根据行业属性确定基础PE：

| 行业类型 | 基础PE | 理由 |
|----------|--------|------|
| 必选消费（白酒、调味品、乳制品） | 20x | 需求稳定，现金流可预期 |
| 可选消费（家电、汽车、服装） | 15x | 周期性略强 |
| 医药（创新药、CXO） | 20x | 长周期成长 |
| 医药（仿制药、中药） | 15x | 成长性有限 |
| 科技（互联网、软件） | 20x | 高成长但波动大 |
| 科技（硬件、半导体） | 18x | 周期+成长 |
| 金融（银行） | 8x | 高杠杆、低增速 |
| 金融（保险、券商） | 12x | 周期性强 |
| 周期（钢铁、煤炭、化工） | 8x | 强周期，利润波动大 |
| 公用事业（电力、水务） | 12x | 稳定但低增速 |
| 地产 | 6x | 行业下行周期 |

如果行业不在列表中，根据行业特征类比选取。

#### 3.2 加分因子

| 因子 | 条件 | 加分 |
|------|------|------|
| 护城河-品牌 | 毛利率>70%且5年波动<3% | +5x |
| 护城河-品牌 | 毛利率>50%且5年波动<5% | +3x |
| 护城河-规模 | 营收为行业第二名2倍以上 | +3x |
| 护城河-规模 | 营收为行业第一 | +2x |
| ROE优秀 | ROE>25%且近3年逐年提升 | +3x |
| ROE优秀 | ROE>20%且近3年稳定 | +2x |
| ROE良好 | ROE>15%且近3年稳定 | +1x |
| 现金流优秀 | 经营现金流/净利润>1（近3年均值） | +2x |
| 现金流良好 | 经营现金流/净利润>0.8（近3年均值） | +1x |
| 高成长 | 近3年净利CAGR>20% | +3x |
| 稳定成长 | 近3年净利CAGR 10%-20% | +2x |
| 分红优秀 | 股息率>3%且连续5年以上 | +2x |
| 分红良好 | 股息率>2%且连续3年以上 | +1x |

#### 3.3 减分因子（财务风险）

| 因子 | 条件 | 减分 |
|------|------|------|
| 增速下滑 | 最新增速<近3年均值的50% | -3x |
| 增速下滑 | 最新增速<近3年均值的70% | -2x |
| 负增长 | 最新营收或净利负增长 | -3x |
| 现金流恶化 | 经营现金流/净利润<0.5 | -2x |
| 高负债 | 资产负债率>60%（非金融） | -2x |
| 应收异常 | 应收增速>营收增速×1.5 | -2x |
| 利润率下滑 | 净利率连续2年下降 | -1x |
| ROE下滑 | ROE连续2年下降 | -1x |

#### 3.4 减分因子（行业/商业模式风险）

根据行业匹配预设风险因子，再结合具体公司微调：

**白酒行业预设风险：**
| 风险 | 减分 | 说明 |
|------|------|------|
| 消费人群萎缩 | -2x | 人口负增长+年轻人饮酒减少，不可逆 |
| 政策风险 | -2x | 消费税改革、限制三公消费 |
| 消费场景依赖 | -1x | 政商宴请占比高 |
| 替代品竞争 | -1x | 低度酒、洋酒侵蚀 |
| 品类集中 | -1x | 核心大单品收入占比>60%时触发 |
| 渠道库存风险 | -1x | 经销商模式，库存周期放大波动 |

**医药行业预设风险：**
| 风险 | 减分 | 说明 |
|------|------|------|
| 集采政策 | -3x | 仿制药集采持续推进 |
| 医保控费 | -2x | 医保谈判持续压价 |
| 研发失败 | -2x | 创新药管线不确定性 |
| 审批风险 | -1x | 政策审批周期长 |
| 技术迭代 | -1x | 新疗法可能颠覆现有产品 |

**科技行业预设风险：**
| 风险 | 减分 | 说明 |
|------|------|------|
| 技术迭代快 | -2x | 护城河可能被颠覆 |
| 监管风险 | -2x | 反垄断、数据安全 |
| 客户集中 | -1x | 大客户依赖度>30%时触发 |
| 人才流失 | -1x | 核心技术人员依赖 |

**地产行业预设风险：**
| 风险 | 减分 | 说明 |
|------|------|------|
| 行业下行周期 | -3x | 需求长期收缩 |
| 政策压制 | -2x | 房住不炒 |
| 高杠杆 | -2x | 行业普遍高负债 |
| 流动性风险 | -2x | 债务到期压力 |

**周期行业预设风险：**
| 风险 | 减分 | 说明 |
|------|------|------|
| 强周期波动 | -2x | 利润波动幅度大 |
| 产能过剩 | -2x | 供给端竞争激烈 |
| 政策调控 | -1x | 环保、产能限制 |

**金融行业预设风险：**
| 风险 | 减分 | 说明 |
|------|------|------|
| 资产质量 | -2x | 坏账/信用风险不透明 |
| 利率风险 | -1x | 净息差收窄 |
| 监管变化 | -1x | 资本充足率等要求趋严 |

**其他行业：** 根据行业特征动态判断，给出2-5项核心风险，每项-1x到-3x。

#### 3.5 公司特有风险微调

在行业预设基础上，根据具体公司情况微调：
- 如果公司在某项风险上明显优于行业（如茅台受替代品竞争影响最小），可减轻该项减分
- 如果公司有行业预设之外的特殊风险（如管理层变动、诉讼等），额外减分
- 微调时必须给出理由

#### 3.6 合理PE计算

```
合理PE = 行业基础PE + 加分合计 + 减分合计
```

**下限保护**：合理PE不低于8x（除非公司基本面严重恶化）
**上限保护**：合理PE不超过50x（防止过于乐观）

### Step 4: 多模型估值与交叉验证

不同企业特征适用不同估值模型。先判断企业类型，再选择对应的模型组合。

> ⚠️ **重要：本节是 Step 0 判完类后的"查表确认"，不是独立判类**。
> Step 0 已经基于 [估值方法选择框架](../../wiki/strategies/估值方法选择框架.md) 5 问决策树定下了主估值方法（PE / PB / P/S / PEG），本节的 7 类企业表只是用于：
> 1. 确认 Step 0 选定的主估值与本节一致
> 2. 在 PE 主路径下，进一步细分到"利润改善型/高分红型/困境反转型"以微调因子
>
> **不要在本节重新判类**——若与 Step 0 矛盾，回到 Step 0 重判，不要在 Step 4 自行改变主估值。

#### 4.1 企业分类与模型选择

| 企业类型 | 特征 | 主模型 | 辅助模型 | DCF权重 |
|----------|------|--------|---------|---------|
| **现金牛型** | 现金流稳定、增速平稳、商业模式成熟（如茅台、长江电力） | PE定价 | DCF（高可靠性） | 与PE等权，取均值 |
| **利润改善型** | 现金流波动但利润率持续提升（如青啤、海天味业转型期） | PE定价 | DCF（中可靠性） | 仅作参考，不参与定价，但若DCF远低于PE模型需标注风险 |
| **高成长型** | 营收高增速（>25%）、利润可能尚未释放（如早期新能源） | PEG定价 | PE定价 | 不适用（现金流不稳定） |
| **真周期型** | 营收+销量同步周期波动（钢铁/煤炭/水泥/有色等大宗品） | PB定价 | PE仅在周期底部参考 | 不适用（FCF波动太大） |
| **弱周期成长型** ★ | **营收单边上行 + 利润率周期波动**（汽车/家电/部分制造）| **P/S定价** | 中周期PE（营收×正常化净利率×PE）+ SOTP | 不适用（FCF波动）|
| **高分红型** | 低增速但分红稳定（如银行、高速公路） | DDM定价 | PE定价 | DCF作为补充 |
| **困境反转型** | 当前亏损或微利，但有明确反转预期 | 目标利润×合理PE | — | 不适用 |

> ⚠️ **"真周期型" vs "弱周期成长型" 的关键区别**：
> - 真周期：营收+销量同步周期波动，萎缩年常见 → PB 主
> - 弱周期成长：营收单边上行（无萎缩年），仅利润率受行业内卷扰动 → **P/S 主**（汽车/家电请用此类）
> - 长城汽车 2026-04-28 案例：营收 8 年 +125% 单边上行（仅 2019 -3%），错归"周期型"用 PE/PB 会双重打折，应归"弱周期成长型"用 P/S

#### 4.2 各模型计算方法

##### PE模型（所有类型通用）

```
合理价值_PE = 合理PE × TTM EPS
```

合理PE由 Step 3 的因子模型推导。

##### DCF模型（现金牛型/高分红型适用）

两阶段自由现金流折现：

```python
# 参数选择原则（保守为主）
# 自由现金流 = 经营现金流 - 资本支出
# 5年增长率：取近3年FCF增速和净利增速的较低者，再打8折
# 永续增长率：消费品2-3%，周期品1-2%，公用事业1-2%
# 折现率：
#   高确定性企业 8-9%（茅台、长电等）
#   中确定性企业 9-10%
#   低确定性企业 10-12%

def dcf(fcf, growth_5y, terminal_growth, discount_rate, shares):
    # 阶段1：5年预测期
    pv_sum = sum(fcf * (1+growth_5y)**i / (1+discount_rate)**i for i in range(1,6))
    # 阶段2：永续价值
    terminal_fcf = fcf * (1+growth_5y)**5 * (1+terminal_growth)
    terminal_pv = terminal_fcf / (discount_rate - terminal_growth) / (1+discount_rate)**5
    return (pv_sum + terminal_pv) / shares
```

**必须附敏感性分析**：折现率±1%、永续增长率±1%的3×3矩阵，展示估值对参数的敏感度。

##### DDM模型（高分红型适用）

```
合理价值_DDM = 下一年预期股息 / (要求回报率 - 股息增长率)

# 要求回报率：8-10%
# 股息增长率：取近5年股息CAGR，上限不超过净利增速
```

##### PEG模型（高成长型适用）

```
合理PE_PEG = 净利增速(%) × PEG倍数
# 保守PEG = 0.8，合理PEG = 1.0
合理价值_PEG = 合理PE_PEG × TTM EPS
```

##### PB模型（真周期型 / 金融型适用）

```
# 用历史PB区间定价
# 买入价 = 历史低点PB中位数 × 每股净资产
# 卖出价 = 历史高点PB中位数 × 每股净资产
```

##### P/S模型（弱周期成长型适用）★

> 详见 [估值方法选择框架](../../wiki/strategies/估值方法选择框架.md) 的 P/S 推导通用模板和子行业基准。

```
合理 P/S = 全球可比龙头基准 P/S
        ± 规模因子（小公司 -0.10~-0.20 / 巨头 +0.10）
        ± 成长因子（出海+高端化等结构变化 +0.05~+0.15）
        ± 净利率因子（短期承压 -0.10 / 长期高于同行 +0.10）
        ± 治理 / 公司特有因子 ±0.05

合理价值_PS = 合理 P/S × 中周期营收（前瞻 1-2 年）/ 总股本
```

**强制要求**：

1. **同行可比表**：列出全球 + 国内 5-8 家可比公司的 P/S（市值/营收），并给出本公司在分布中的位置。优先使用 [估值方法选择框架](../../wiki/strategies/估值方法选择框架.md) 的子行业 P/S 基准；wiki 没列的子行业必须自行实证补充
2. **历史 P/S 中枢检查**：5-10 年区间散点，标注峰谷年份并**剔除主题泡沫年份**（如汽车股 2020-2021 PE 60x 等极端值）
3. **三模型加权**：P/S 50% + 中周期 PE 30% + SOTP 20%（详见 4.3）

##### 中周期 PE 模型（弱周期成长型辅助）

```
# 区别于 Step 3 的合理 PE 因子推导：
# 中周期 PE 用"营收 × 正常化净利率 × 合理 PE / 总股本"
# 避免 TTM PE 在利润率底位时的双重打折

合理价值_中周期PE = 中周期营收 × 正常化净利率 × 合理 PE / 总股本

# 正常化净利率：取历史 5-10 年中位数，避开峰谷年份
# 合理 PE：弱周期成长股给 14-15x（vs 真周期股 8-10x）
```

##### SOTP 分部估值（弱周期成长型 + 多业务多市场公司辅助）

```
# 按业务/市场拆分，每块给不同 P/S 或 PE
# 示例（长城汽车）：
# 国内主流 营收 × P/S 0.5x（同长安）
# 国内高端 营收 × P/S 1.5x（同理想）
# 海外     营收 × P/S 1.2x（出海溢价）
# 合计 → 总市值 / 总股本 → 合理价
```

#### 4.3 交叉验证规则

根据企业类型，对多模型结果交叉验证：

**现金牛型**（PE + DCF 等权）：
```
综合合理价值 = (合理价值_PE + 合理价值_DCF) / 2
```
如果两个模型偏差>30%，需要检查假设是否合理并在报告中说明分歧原因。

**利润改善型**（PE为主，DCF参考）：
```
综合合理价值 = 合理价值_PE
# 但如果 DCF估值 < 当前股价，标注：
# "DCF估值(XX元)低于当前价，说明当前价格隐含了对未来利润率持续改善的预期，若改善不及预期存在下行风险"
```

**高成长型**（PEG为主，PE校验）：
```
综合合理价值 = 合理价值_PEG
# 若 合理价值_PE < 合理价值_PEG × 0.7，说明PE模型认为高成长不可持续，标注风险
# 注意：使用PEG为主模型时，增速已决定了合理PE，Step 3中的增长类加分因子（高成长+3x、稳定成长+2x）
# 在静态PE中应剔除，避免增速被双重计算。仅保留与增速无关的因子（护城河、现金流、风险等）。
```

**真周期型**（PB为主）：
```
综合合理价值 = 基于PB区间
# PE仅在行业周期底部（利润极低时）作为辅助参考
```

**弱周期成长型**（P/S 主 + 中周期 PE 辅 + SOTP 辅 三模型加权）★：

```
综合合理价值 = 合理价值_PS × 50%
            + 合理价值_中周期PE × 30%
            + 合理价值_SOTP × 20%
```

> ⚠️ **权重 + 阈值依据声明（双重局限）**：
> - **权重 50/30/20** 基于长城汽车 v3 单案例校准（与市场实际定价偏离 -16%）
> - **复查阈值 30% / 50%** 同样基于单案例拍脑袋
> - **两者都需要 3-5 个标的回测后再调**
>
> 在精度未稳定前，**真正起兜底作用的是"差异是否引发复查"这一行为**，而不是阈值本身的精度——若 P/S 与 PE 差距让你"感到不舒服"，无论是 25% 还是 35%，都应该复查。不必拘泥精确比例。

**复查规则（比权重精度更重要）**：

| 触发条件 | 行动 |
|---------|------|
| **P/S 与中周期 PE 差距 > 30%**（阈值待回测）| **优先信 P/S**（营收锚比净利率锚稳定）|
| **P/S 与中周期 PE 差距 > 50%**（阈值待回测）| **强制复查**：(a) 同行可比表是否选错对标？(b) 净利率假设是否过激进/保守？(c) SOTP 各分部 P/S 是否合理？|
| **三模型最终结果 vs 市场实际定价偏离 > ±20%** | 触发反向健康检查（Step 0 末尾），重新走决策树 |

**校准记录**（每新增一个弱周期成长股案例后更新）：
- 长城汽车 v3：P/S 25 / PE 22 / SOTP 26 → 加权 24 元 → vs 市场 20.23 = **-16% 偏离**（合格）
- 待补：第 2-5 个案例（美的/立讯/比亚迪等）回测后调整权重

**高分红型**（DDM + PE 等权）：
```
综合合理价值 = (合理价值_DDM + 合理价值_PE) / 2
# 注意：使用DDM为主模型时，分红已决定了定价基础，Step 3中的分红类加分因子（分红优秀+2x、分红良好+1x）
# 在静态PE中应剔除，避免分红贡献被双重计算。仅保留与分红无关的因子（护城河、成长性、风险等）。
```

### Step 4B: 远期分析（1-2年前瞻估值）

静态PE模型用的是TTM/最近年报EPS，对于业务结构正在变化的公司（如AI转型、新业务放量、补贴退出）会严重失真。远期分析通过分业务建模，估算1-2年后的利润水平和合理PE，折现回当前与静态模型交叉验证。

**适用场景**：当以下任一条件成立时，必须做远期分析：
- 公司正在经历业务结构显著变化（新业务收入占比快速提升或下降）
- 存在已知的临时性费用/收入（闪购补贴、一次性减值、并购整合费用）
- 核心业务增速与新业务增速方向相反（如电商放缓但云加速）
- 当前PE因临时因素显著偏离合理PE

**不适用**：
- 业务结构稳定、增速平稳的成熟企业（如茅台、长江电力），静态PE足够
- **真周期型企业**（如煤炭、钢铁、猪肉、大宗化工）：利润随行业周期大幅波动，1-2年的线性外推会严重失真。周期股应使用PB定价+周期位置判断，不做远期利润预测

**特别适用**：
- **弱周期成长型企业**（汽车/家电/部分制造）：营收单边上行但利润率周期波动，远期分析必须做——但**重点是营收建模而非利润，配合 Step 4 的 P/S 主估值**

**前置判断流程**：进入远期分析前，先确认企业类型（Step 0）。如果是**真周期型**，跳过整个Step 4B，直接进入Step 5。

#### 4B.1 分业务营收建模

将公司营收拆解为主要业务板块（3-5个），逐个估算未来2个财年的增速：

```
| 业务板块 | 当前营收 | 占比 | FY+1增速 | FY+2增速 | 依据 |
|----------|---------|------|---------|---------|------|
| 板块A | xxx亿 | xx% | x% | x% | {具体理由} |
| 板块B | xxx亿 | xx% | x% | x% | {具体理由} |
| ... | | | | | |
| 合计 | xxx亿 | 100% | x% | x% | |
```

**增速估算原则（保守为主）：**
- 成熟业务：取近2年增速的较低值，除非有明确拐点信号
- 高增长业务：取管理层指引或分析师一致预期的80%（打折）
- 下滑业务：按最近2个季度的同比趋势线性外推
- 新业务（<2年历史）：不给高增速假设，按行业平均增速

#### 4B.2 利润率预测

对每个业务板块预测目标利润率：

```
| 业务板块 | 当前OPM | FY+1 OPM | FY+2 OPM | 依据 |
|----------|--------|---------|---------|------|
| 板块A | xx% | xx% | xx% | {对标/趋势/天花板分析} |
| 板块B | xx% | xx% | xx% | |
| 集团费用 | -xx亿 | -xx亿 | -xx亿 | {总部/股权激励等} |
```

**利润率估算原则：**
- 改善中的业务：参照同行业成熟公司的利润率作为天花板，不超过天花板的90%
- 稳定业务：维持当前水平
- 恶化中的业务：按最近趋势外推，除非有明确的成本削减计划
- 临时性费用（如闪购补贴）：基准情景假设退出，悲观情景假设持续

#### 4B.3 远期EPS计算

```python
# 远期经营利润 = Σ(各板块营收 × 各板块OPM) - 集团费用
# 远期净利润 = 远期经营利润 × (1 - 有效税率) + 预计投资收益（保守取0或近3年均值×50%）
# 远期EPS = 远期净利润 / 预计股本数（考虑回购）
```

**注意**：
- 投资收益/损失波动大的公司（如阿里、腾讯），远期净利润估算中投资收益取保守值（0或历史均值的50%）
- 有回购计划的公司，按近2年回购速度估算未来股本减少
- 有大额股权激励(SBC)的公司（科技公司常见），需估算每年SBC导致的股本净增加：
  净股本变化 = 回购减少的股本 - SBC新增的股本
  远期股本 = 当前股本 × (1 + 年净变化率)^N
  如果SBC占营收比例>5%，应在经营利润中扣除SBC费用（如果财报未扣除）

#### 4B.4 因子调整SOTP PE

远期合理PE不能简单用行业通用PE加权——那会丢失公司特有的质量因子（护城河、现金流、风险）。正确做法是先算原始SOTP，再逐项审查静态因子是否仍然适用。

##### 第一步：增速调整SOTP PE

每个板块的基础PE应根据该板块的预期增速进行调整。高增速板块理应获得更高PE，低增速板块则打折。

```
各板块远期PE = 行业基础PE × 增速调整系数

增速调整系数（线性插值公式）：
- <3%: 系数 = 0.8
- 3%-25%: 系数 = 0.8 + (growth - 0.03) / 0.22 × 0.7  （从0.8线性增长到1.5）
- >25%: 系数 = 1.5（封顶）

参考区间表（实际计算用上述线性插值，此表仅供快速参考）：
| 板块预期增速(FY+1~FY+2均值) | 参考系数 | 理由 |
|---------------------------|---------|------|
| <3% (近乎零增长) | ×0.8 | 价值消耗 |
| 3-8% (低增长) | ×0.8~1.0 | 行业平均 |
| 8-15% (稳健增长) | ×1.0~1.2 | 高于行业 |
| 15-25% (高增长) | ×1.2~1.5 | 增长溢价 |
| >25% (超高增长) | ×1.5 | 高成长溢价，但PE上限50x |

增速调整SOTP PE = Σ(各板块FY+2营收占比 × 各板块增速调整后PE)
```

**原则**：调整系数反映的是增速对估值倍数的影响，不是对利润的影响（利润已在EPS中）。一个增速18%的广告业务和增速3%的本地生活，即使利润率相同，前者也应该获得更高PE。

##### 第二步：因子存续性审查

逐项检查 Step 3 中的每个加分/减分因子，判断其在远期分业务模型中是否已被"吸收"：

| 吸收状态 | 含义 | 处理方式 |
|----------|------|---------|
| **已吸收** | 因子效果已体现在板块增速或利润率预测中 | 不重复计入 |
| **部分吸收** | 因子效果部分体现在板块指标中，但有超越板块指标的公司层面溢价 | 按50%折扣计入 |
| **未吸收** | 因子完全是公司层面的系统性优势/风险，无法通过板块指标体现 | 全额计入 |

**常见因子的典型判断：**

| 因子类型 | 典型吸收状态 | 理由 |
|----------|------------|------|
| ROE优秀 | 已吸收 | ROE高是因为利润率高+资产效率高，这些已经在板块OPM和增速中体现 |
| 高成长/稳定成长 | 已吸收 | 增速已经是板块预测的输入变量 |
| 毛利率护城河 | 已吸收 | 毛利率水平已在板块OPM中体现 |
| 生态协同/网络效应 | 部分吸收 | 对各板块增速有贡献（已吸收），但跨板块协同、用户锁定效应超越单一板块指标（未吸收部分） |
| 规模优势 | 部分吸收 | 市场份额体现在营收（已吸收），但规模带来的议价能力和壁垒超越营收数字（未吸收部分） |
| 现金流优秀 | 部分吸收 | 利润率高意味着现金流基础好（已吸收），但OCF/NI>1的运营资本管理能力是公司层面的（未吸收部分） |
| 分红优秀 | 未吸收 | 分红政策是公司决策，不在板块指标中 |
| 监管风险 | 部分吸收 | 对增速的压制已在板块增速中考虑（已吸收），但黑天鹅式政策冲击无法用增速预测覆盖（未吸收部分） |
| 地缘政治风险 | 已吸收 | 板块基础PE使用的是中国公司估值水平，已内嵌地缘折价。若局势进一步恶化，应调整板块基础PE而非叠加额外因子 |
| 行业竞争加剧 | 已吸收 | 竞争影响已反映在板块增速和利润率预测中 |

##### 第三步：计算因子调整后SOTP PE

输出格式：

```
| 静态因子 | 静态PE贡献 | 吸收状态 | 远期PE贡献 | 理由 |
|----------|-----------|---------|-----------|------|
| 网络效应 | +3x | 部分吸收(50%) | +1.5x | 跨板块协同超越单一板块 |
| 规模优势 | +3x | 已吸收 | 0x | 市场份额已在营收中 |
| ROE稳定 | +2x | 已吸收 | 0x | 已在利润率中 |
| 监管风险 | -2x | 部分吸收(50%) | -1x | 黑天鹅部分未在增速中 |
| 地缘政治 | -2x | 未吸收 | -2x | 系统性外部风险 |
| ... | | | | |
| 合计调整 | | | +Nx / -Nx | |

因子调整后SOTP PE = 原始SOTP PE + 未吸收因子合计
```

**与静态PE的关系**：
- 如果因子调整后SOTP PE与静态PE差异<3x → 两个模型一致性好，可互相验证
- 如果差异>5x → 需要检查：是因子模型"质量溢价堆叠"过度，还是SOTP遗漏了重要因子
- 一般来说，因子调整后SOTP PE应低于静态因子PE（因为部分因子被吸收），但不应低太多（>8x差距说明有问题）

#### 4B.5 远期目标价与隐含回报率

```
远期目标价 = 远期合理PE × 远期EPS
当前价隐含年化回报率 = (远期目标价/当前价)^(1/N) - 1  # N=1或2年

# 加上预期分红：
含息年化回报率 = 隐含回报率 + 当前股息率
```

**回报率判断标准：**

| 含息年化回报率 | 判断 | 对买入决策的影响 |
|---------------|------|----------------|
| >20% | 显著低估 | 可直接买入，即使静态模型未到安全边际买入价 |
| 15-20% | 有吸引力 | 值得建仓，但控制仓位 |
| 10-15% | 合理回报 | 持有，但不追加 |
| 5-10% | 回报偏低 | 谨慎，寻找更好标的 |
| <5% | 无吸引力 | 不建议买入，已持有考虑减仓 |

#### 4B.6 与静态模型交叉验证

```
| 估值方法 | 合理价值/目标价 | 权重 | 说明 |
|----------|---------------|------|------|
| 静态PE模型 | xx元 | 参考 | Step 4 结果 |
| 远期估值(FY+1) | xx元 | 主要 | 远期PE × 远期EPS，折现1年 |
| 远期估值(FY+2) | xx元 | 辅助 | 远期PE × 远期EPS，折现2年 |
| DCF模型 | xx元 | 参考 | Step 4 结果 |
```

**验证规则**：
- 如果远期估值显著高于静态估值(>20%)：说明当前利润是低点，公司在"蓄力"期，静态PE低估了公司 → 综合估值上修
- 如果远期估值显著低于静态估值(>20%)：说明当前利润是高点或不可持续 → 综合估值下修，提示风险
- 如果两者接近(±20%)：业务结构稳定，静态模型可靠

**综合估值（加入远期分析后）**：
```
# 有远期分析时：
综合合理价值 = 静态合理价值 × 40% + 远期FY+1折现价值 × 40% + 远期FY+2折现价值 × 20%

# 折现率 = Step 4 DCF中使用的折现率
```

**方法**：用 Step 1B 的利润瀑布图作为基线，对每个P&L行项独立估算 FY+1（表格结构同 Step 1B.2，增加"FY+1E"和"假设依据"列）。核心税率沿用 Step 1B.4 的计算值。

**逐行估算要点：**

- **管理费用**：单独估算内含的 SBC 变化
- **财务费用**：分拆利息收支和汇兑损益，不能简单外推（如药明康德2024→2025从-7.87→+3.98亿）

### Step 5: 预期差评估

市场下跌的核心驱动力不是"基本面差"，而是"基本面比预期差"。同样的6%增速，如果市场原本预期18%，PE会暴跌；如果市场原本就预期5%，PE反而可能稳住。安全边际的宽窄应该反映这种预期差风险。

#### 5.1 判断当前PE隐含的增长预期

将当前PE与不同增速假设下的"合理PE"对比，反推市场在定价什么水平的增长：

```
当前PE对应的隐含增速 ≈ 查看合理PE模型中，需要什么增速假设才能得到当前PE

示例（茅台）：
- 当前PE 20x，模型中增速18%时合理PE=29x，增速6%时合理PE=22x
- 当前20x < 22x → 市场甚至在定价比6%更差的预期
- 说明悲观预期已经充分Price in
```

#### 5.2 逐季度增速趋势分析

不只看最新一期增速，而是看趋势方向：

```python
# 提取逐季度累计增速，识别趋势
# 关键：拆出单季度增速（累计值相减），判断是加速、稳定还是减速

# 趋势判断标准：
# - 加速：最近2个单季度增速逐季上升
# - 稳定：最近3个单季度增速波动<3个百分点
# - 减速：最近2个单季度增速逐季下降
# - 失速：最新单季度增速<3%或转负
```

#### 5.3 预期差分类

综合当前PE位置和增速趋势，判断预期差状态：

| 状态 | 条件 | 含义 | 对安全边际的影响 |
|------|------|------|----------------|
| **预期过高** | 当前PE > 合理PE × 1.1，且增速在减速 | 市场还没反应过来，下行风险大 | 安全边际比例 **+10%**（更严格） |
| **预期合理** | 当前PE 在合理PE ±10%范围内 | 市场定价基本到位 | 安全边际比例 **不调整** |
| **预期偏低** | 当前PE < 合理PE × 0.9，且增速稳定或加速 | 市场过度悲观 | 安全边际比例 **-5%**（可适当放宽） |
| **预期已崩** | 当前PE < 合理PE × 0.8，增速减速但未失速 | 最坏预期已充分反映 | 安全边际比例 **-5%**，且标注"悲观预期已Price in，进一步大跌概率较低" |
| **基本面恶化** | 增速失速（单季<3%或负增长），且趋势未见拐点 | 不确定是否见底 | 安全边际比例 **+5%**，并标注"趋势未明，建议观望或小仓试探" |

#### 5.4 预期差调整示例

**茅台2024年初（回测）：**
- 当时PE约30x，合理PE 29x → 预期合理
- 但增速处于减速初期（尚未反映在PE中）
- 判定：**预期过高**（PE还在定价18%增速，实际已开始减速）
- 安全边际 +10% → 从20%变为30%

**茅台当前（2026年4月）：**
- 当前PE 20x，合理PE 22x → 当前PE < 合理PE × 0.9
- 增速减速中（25Q3单季仅0.3%），但属于"失速"级别
- 判定：**基本面恶化** vs **预期已崩** 的边界
- 如果认为0.3%是季节性/一次性 → 预期已崩，安全边际-5%
- 如果认为是趋势性恶化 → 基本面恶化，安全边际+5%
- **需要4月17日年报Q4数据来判断** → 当前暂按"预期偏低"处理，安全边际不调整，但标注不确定性

#### 5.5 市场定价校验

预期差评估关注的是"当前PE vs 合理PE"的瞬时偏差。但如果市场长期给出的PE中枢远低于模型合理PE，需要反思模型是否偏乐观。

**校验方法：计算近半年PE中枢，与模型合理PE对比**

```python
import yfinance as yf

ticker = yf.Ticker('代码')
hist = ticker.history(period='2y')

# 近半年PE中枢
recent_6m = hist.tail(120)  # 约半年交易日
recent_avg_price = recent_6m['Close'].mean()
recent_pe = recent_avg_price / ttm_eps

# 近2年PE分布（按PE区间统计停留天数占比）
```

**校验流程：发现分歧 → 分析原因 → 判断谁对 → 决定修正**

**第一步：量化分歧程度**

| 市场PE中枢 vs 模型合理PE | 分歧程度 |
|-------------------------|---------|
| 偏差在 +/-15% 以内 | 无显著分歧，跳过后续分析 |
| 偏差 15-25% | 中度分歧，需分析原因 |
| 偏差 25% 以上 | 严重分歧，必须深入分析 |

**第二步：分析市场定价的原因（核心步骤）**

发现分歧后，不能机械修正，必须回答：**市场为什么给这个PE？市场看到了什么模型没捕捉到的？**

需要分析的维度：

1. **利润率改善空间分析**（针对利润改善型企业）
   - 毛利率提升空间：对标行业天花板（如啤酒行业重庆啤酒高端化成熟期毛利率~48%）
   - 费用率压缩空间：当前费用率是否已到合理下限
   - 净利率天花板：参照同行最优水平，估算还剩多少提升空间
   - 剪刀差趋势：营收增速 vs 净利增速的差值是否在收窄（收窄意味着利润率改善的加成效应在减弱）

2. **增长可持续性分析**
   - 营收增长是靠量还是价？量增逻辑是否见顶？
   - 提价空间还有多大？消费者对价格敏感度如何？
   - 行业总量是增长、平稳还是萎缩？

3. **可比公司定价参照**
   - 同类型（零增长+高分红）公司市场给多少PE？
   - 银行股（零增长+高分红）：5-7x
   - 公用事业（零增长+稳定分红）：10-12x
   - 消费品（低增长+品牌溢价）：15-20x
   - 消费品（稳定增长）：20-25x
   - 当前企业更像哪一类？

**第三步：判断谁对并决定修正**

| 分析结论 | 修正方式 | 示例 |
|----------|---------|------|
| **市场合理**：模型加分项过于乐观，或未充分反映增长放缓/天花板 | 合理PE直接采用市场中枢PE（±1x微调） | 青啤：利润率改善接近天花板，营收零增长，市场18-19x定价合理 → 合理PE采用19x |
| **市场略悲观**：基本面没那么差，但市场担忧有一定道理 | 合理PE = (模型PE + 市场PE) / 2 | 某公司短期利空但长期逻辑未破 |
| **市场显著悲观**：市场因短期恐慌过度杀跌，基本面不支持这么低的PE | 合理PE保持模型值，不修正 | 2024年茅台PE杀到20x，但品牌和盈利能力未受损 |
| **市场偏贵**：市场PE高于模型 | 合理PE保持模型值（本模型偏保守，不上修） | 牛市中消费股PE普遍偏高 |

**关键原则：修正必须附理由。不允许无分析的机械修正。**

**第四步：修正联动**

修正合理PE后，以下全部重新计算：
- 合理价值
- 安全边际买入价（锚点A和锚点B上限）
- 减仓价和清仓价

**示例（青岛啤酒）：**
- 模型合理PE: 24x，市场近半年PE中枢: 18.5x，偏差-23% → 中度分歧
- 原因分析：
  - 净利率从8%→14%，但对标重庆啤酒天花板约18%，剩余空间仅3-4个百分点
  - 费用率已从22%压至17%，继续压缩空间有限
  - 剪刀差从+35%收窄至+4%，利润率改善加成效应接近耗尽
  - 营收零增长，一旦净利率见顶，利润增速≈营收增速≈0%
  - 定价参照：零增长消费品公司通常15-20x
- 结论：**市场合理**，18-19x充分反映了"增长故事接近尾声"的现实
- 修正：合理PE从24x修正为 **19x**（采用市场中枢，品牌溢价给+0.5x取整）

预期差评估的输出格式见末尾"## 输出格式"统一模板。

### Step 6: 安全边际计算

**重要说明**：如果执行了远期分析（Step 4B），则 Step 6、Step 7、Step 8 中所有提到的"综合合理价值"统一替换为 Step 4B.6 的综合估值结果（即含远期分析的加权综合估值），而非 Step 4 的纯静态估值。

安全边际不能只用合理价值打固定折扣，必须结合历史估值区间做现实性校验。一个永远到不了的买入价没有实操意义。

#### 6.1 获取历史估值区间

用 yfinance 拉取近5-10年的年度最低价，结合当年EPS计算历史低点PE：

```python
import yfinance as yf

ticker = yf.Ticker('600600.SS')
hist = ticker.history(period='10y')

# 按年取最低收盘价
for year in range(2016, 2027):
    yearly = hist[hist.index.year == year]
    if len(yearly) > 0:
        print(f"{year}: 最低 {yearly['Close'].min():.2f}  最高 {yearly['Close'].max():.2f}")
```

然后用当年EPS计算各年低点PE和高点PE，得出：
- **历史低点PE区间**：取近5年低点PE的中位数
- **历史高点PE区间**：取近5年高点PE的中位数

#### 6.2 三锚定价法

安全边际买入价由三个锚点综合决定：

| 锚点 | 计算方式 | 含义 |
|------|---------|------|
| 锚点A：固定折扣法 | 综合合理价值 × (1 - 安全边际比例) | 基于内在价值的理论买入价 |
| 锚点B：历史低点PE法 | 历史低点PE中位数 × 当前TTM EPS | 市场实际给过的低估值 |
| 锚点C：悲观情景法 | 综合合理价值 × 0.7 | 考虑极端下行情况 |

**定价规则：**

```
# 锚点B受最低折扣约束，防止安全边际形同虚设
锚点B上限 = 综合合理价值 × (1 - 最低折扣)
有效锚点B = min(锚点B原始值, 锚点B上限)

安全边际买入价 = max(锚点A, 有效锚点B)
```

**最低折扣**（锚点B的硬约束）：

| 确定性等级 | 最低折扣 | 含义 |
|-----------|---------|------|
| 高确定性 | 10% + 预期差调整 | 预期过高时+10%，预期偏低时-5% |
| 中确定性 | 15% + 预期差调整 | 同上 |
| 低确定性 | 20% + 预期差调整 | 同上 |

预期差调整量来自 Step 5 的预期差评估结果。最低折扣下限为5%（即使预期已崩也保留最少5%折扣）。

**取值逻辑：**
- 锚点A是理论最优买入价，基于固定折扣
- 有效锚点B用历史数据做现实性校验，但不允许突破最低折扣门槛
- 取两者较高者，兼顾安全边际充足性和价格可达性
- 锚点C仅作参考展示，不参与定价

**示例（青岛啤酒，中确定性）：**
- 合理价值 74元，锚点B原始值 = 20.8x × 3.36 = 70元
- 锚点B上限 = 74 × 0.85 = 63元（最低折扣15%）
- 有效锚点B = min(70, 63) = 63元
- 锚点A = 74 × 0.75 = 55元
- 安全边际买入价 = max(55, 63) = **63元**（而非之前的70元）

#### 6.3 安全边际比例（用于锚点A）

根据企业确定性浮动：

| 确定性等级 | 安全边际 | 适用条件 |
|-----------|---------|---------|
| 高确定性 | 20% | 合理PE加分项>=10x，且无-3x的单项减分 |
| 中确定性 | 25% | 默认级别 |
| 低确定性 | 35% | 减分合计绝对值>加分合计，或有多个-3x减分 |

#### 6.4 盈利趋势修正

对于盈利能力处于明确改善趋势的公司（净利率或ROE连续3年提升），合理PE额外+2x。这反映了当前EPS可能低估了未来的盈利中枢。

#### 6.5 价格计算

```
综合合理价值 = 由 Step 4 多模型交叉验证得出

锚点A = 综合合理价值 × (1 - 安全边际比例)
锚点B = 历史低点PE中位数(近5年) × TTM EPS  （周期型用历史低点PB × 每股净资产）
锚点C = 综合合理价值 × 0.7  # 悲观情景，仅展示不参与定价

安全边际买入价 = max(锚点A, 锚点B)
```

如果有最新季度数据，用年化EPS计算更贴近实际：
- 前三季度数据：年化EPS = 前三季度EPS / (近3年前三季度占全年EPS比例均值)
  示例：白酒Q4占比约30%，则前三季度占比70%，年化EPS = 前三季EPS / 0.70
- 半年报数据：年化EPS = 半年报EPS / (近3年上半年占全年EPS比例均值)

### Step 7: 高估卖出价计算

市场有时会非理性定价（如2021年茅台PE 70x），需要有纪律性地卖出。

#### 7.1 卖出价格计算（双锚校验）

与买入价类似，卖出价也用双锚定价：

| 锚点 | 计算方式 | 含义 |
|------|---------|------|
| 锚点D：溢价倍数法 | 合理价值 × 减仓/清仓倍数 | 基于内在价值的理论卖出价 |
| 锚点E：历史高点PE法 | 历史高点PE中位数(近5年) × TTM EPS | 市场实际给过的高估值 |

**减仓价 = min(锚点D减仓, 锚点E) 附近取值**（取较低者，偏保守）
**清仓价 = 锚点D清仓**（超过合理价值的大幅溢价，纪律性卖出）

#### 7.2 溢价系数

根据企业确定性浮动，高确定性企业可以多给一些溢价空间：

| 确定性等级 | 减仓倍数 | 清仓倍数 | 理由 |
|-----------|---------|---------|------|
| 高确定性 | ×1.3 | ×1.5 | 优质企业可适当容忍溢价，但不追高 |
| 中确定性 | ×1.25 | ×1.4 | 确定性一般，溢价空间更窄 |
| 低确定性 | ×1.2 | ×1.3 | 不确定性大，稍有高估就应离场 |

### Step 8: 投资价值判断

根据当前价格与计算结果的关系，给出明确结论：

| 当前价位 | 结论 | 标签 |
|----------|------|------|
| 当前价 ≤ 安全边际买入价 | 显著低估，安全边际充足 | **买入区间** |
| 安全边际买入价 < 当前价 ≤ 合理价值×0.9 | 低估，但安全边际不足 | **关注区间** |
| 合理价值×0.9 < 当前价 ≤ 合理价值×1.1 | 合理定价 | **持有区间** |
| 合理价值×1.1 < 当前价 ≤ 减仓价 | 偏贵 | **谨慎区间** |
| 减仓价 < 当前价 ≤ 清仓价 | 高估，分批减仓 | **减仓区间** |
| 当前价 > 清仓价 | 显著高估，考虑清仓 | **清仓区间** |

## 区间术语规范（自检）

所有写入 wiki 的区间判定**必须**遵循 `wiki/strategies/价值投资区间判定框架.md`。写 wiki 前对照下列清单自检：

### 标准术语白名单

**个股 6 档**（`wiki/stocks/*.md` 唯一允许的区间术语）：
- `买入区间`、`关注区间`、`持有区间`、`谨慎区间`、`减仓区间`、`清仓区间`
- 位置修饰允许：`XX 区间下沿/上沿/中段/中位`

**分散 ETF / 指数 5 档**（`wiki/funds/*.md` 中指数型 ETF 适用）：
- `深度买入`、`加仓区间`、`持有区间`、`减仓区间`、`卖出区间`

**特殊档位**（必须在该 wiki 明确定义触发条件才可使用）：
- `悲观价`、`极端买入` — 仅针对个别标的（如茅台悲观价 985、中信极端买入 19）

### 禁用词（违规术语）

| 违规术语 | 正确替换 |
|---|---|
| `深度买入`（在个股 wiki 中）| 跌破买入价 × 0.85 必须重做估值；不机械加仓 |
| `加仓区间`（在个股 wiki 中）| `买入区间` |
| `高估区间`、`低估区间` | 从 6 档中选对应项（如 `清仓区间` / `关注区间`）|
| `合理PE区间`（表示价格档位时）| 从 6 档中选对应项；若表示"PE 取值范围"则改写为"**合理 PE 范围**"避免歧义 |
| 无锚点的模糊说法（"有点贵"、"偏低"）| 必须映射到 6 档之一 |

### 写入 wiki 前的自检步骤

1. **确认档位**：当前价是否落在锚点 {买入价/合理价×0.9/合理价×1.1/减仓价/清仓价} 切分的哪一段？
2. **查白名单**：该档位在个股/ETF 的对应术语是否在上表白名单？
3. **grep 违规词**：写入前对该 wiki 文件 grep 禁用词，有则重写
4. **跌破 B×0.85 特殊处理**：若当前价 < 买入价 × 0.85，**禁止**写"深度买入"或自动加仓建议；改为"已跌破买入价 ×0.85，需重做估值（可能是基本面问题）"

### Step 8 区间判定的术语输出

Step 8 的区间标签必须从 6 档白名单中取，不得自创。典型场景示例：

- 当前价 ≤ 买入价 → `买入区间`
- 当前价 ≤ 买入价 × 0.85 → `买入区间（已跌破 B×0.85，建议重做估值）`（而非"深度买入"）
- 当前价 > 清仓价 → `清仓区间`（而非"高估区间"）
- 合理价附近 ±10% → `持有区间`（可加"下沿/上沿"修饰位置）
