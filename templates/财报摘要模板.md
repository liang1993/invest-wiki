# 财报摘要模板

从原始财报 PDF 抽取关键内容到 `raw/reports/<公司>_<报告期>.md`，作为结构化缓存层，避免下次 ingest 时重复解析 PDF。

## 命名规则

- MD 与 PDF 同名：`raw/reports/贵州茅台_2025年年度报告.md` ↔ `贵州茅台_2025年年度报告.pdf`
- 保留 PDF 作为原始凭证，MD 为二次消费的结构化摘要

## 核心原则

1. **每个数据点必须标注 PDF 页码**（格式：`营收1688.38亿(p.6)`），便于审查回查
2. **数字完整保留原单位**（元/亿元/万吨），不做单位换算
3. **直引管理层关键措辞**，用引号标注，不做二次改写
4. **不加工结论/估值判断**，估值分析写在 wiki 页面，摘要仅保留事实

## 模板结构

## 使用流程（集成在 CLAUDE.md ingest 工作流中）

1. **抽取阶段**：读取 PDF 前 20-30 页（通常覆盖第二、三节核心数据），按模板填充，每个数字标注页码
2. **自动生成核对清单**：所有量化数据进入第八章
3. **审查阶段**：按核对清单逐项 `Read PDF pages=X` 回查，更新状态列
4. **发现偏差**：修正摘要 md，不得带偏差进入 wiki
5. **审查通过**：更新 frontmatter `reviewed: true` + `reviewer_note`
6. **回填 wiki**：基于已审查的 md 写入 `wiki/stocks/*.md` 对应章节

## 优点

- PDF 解析成本一次性（抽取+审查），后续消费只需 Read md（1-3 次 tool call vs 原8+次）
- 页码追溯使 wiki 页面的数据可反向追溯到原始财报
- 审查步骤在入库前隔断加工错误
