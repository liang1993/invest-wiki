# 调仓备选库

> 当 focus list 需要调仓时（旧标的减仓 / 现金充裕 / 宏观阶段切换），从本库按触发条件检索候选。本库与 [`wiki/stocks/focus/`](../stocks/focus/) 互补：focus list 是持续追踪仓，备选库是触发条件等待中。

## 用途与定位

- **focus list**：当前长期追踪 + 已估值锚定 + 持续 ingest 财报（13 只）
- **本备选库**：已 deep-dive 建档但暂未入 focus，等触发条件成立后 promote 到 focus（约 22 只）
- **未建档**：偶发 query，无估值锚点，不进本库

→ 本库不是"想买就买的清单"，是 **"宏观信号 / 估值条件 → 触发→ 评估升级到 focus → 决定加仓"** 的一道闸门。

## 三档优先级

| 档位 | 含义 | 触发后操作 |
|---|---|---|
| **A 档** | 触发条件清晰 + 当前估值已贴近买入价（距离 ≤ 10%）| 触发立即评估加入 focus |
| **B 档** | 触发条件需要进一步验证 / 估值偏高需等回撤 | 持续关注，每月仪表盘检查 |
| **C 档** | 纯观察 / 估值远离买入价 / 与现 focus list 高度重叠 | 仅在重大变化时重估 |

## 与 focus list 的协同规则

### 升级路径
```
未建档 → query/财报触发 → deep-dive 建档（wiki/stocks/）
                                        ↓
                                    本备选库
                                        ↓
                            触发条件成立 + 评估通过
                                        ↓
                            链入 wiki/stocks/focus/（加入 focus list）
                                        ↓
                                持续追踪 / 财报 ingest
```

### 降级路径
```
focus 中标的减仓/清仓后失去关注价值
                ↓
        从 wiki/stocks/focus/ 移除符号链接
                ↓
        回流到本备选库（标记降级原因）
                ↓
        若估值再回买入区或基本面变化 → 重新升级
```

### 板块仓位上限（避免重叠加仓）

| 板块 | 当前 focus 持有 | 加仓上限 | 备注 |
|---|---|---|---|
| 金融（银行+保险+券商）| 中信证券、平安银行 | 总仓位 < 25% | 平安加仓需配合平银仓位评估 |
| 互联网平台 | 腾讯、阿里、美团、蜜雪 | 总仓位 < 25% | 已较高，新增需替换 |
| 消费（白酒+啤酒）| 贵州茅台、青岛啤酒 | 总仓位 < 15% | 不建议再加白酒/啤酒 |
| 资源/周期 | 紫金、海螺 | 总仓位 < 15% | 神华/万华加仓需替换 |
| 电信 | 中国移动 | 总仓位 < 10% | 单标的足够 |
| 新能源/汽车 | 宁德 | 总仓位 < 15% | 比亚迪/长城是补充而非替代 |
| 工业自动化 | 汇川 | 总仓位 < 10% | 思源/三一/东方电气是不同子赛道 |

## 操作流程

### 月度仪表盘检查
1. 读 [`wiki/macro/宏观仪表盘.md`](../macro/宏观仪表盘.md) → 确认当前主导阶段
2. 检查本库各阶段触发条件状态：
   - 阶段 ① 已激活
   - 阶段 ② 完成度 X/3
   - 阶段 ③ 完成度 X/3
   - 阶段 ④ 完成度 X/4
   - 阶段 ⑤ AI Capex 周期信号
3. 当某阶段触发条件 ≥ 2/3 满足时 → 重点关注该阶段 A 档候选

### 触发后评估流程
1. 跑 value-invest 重做最新估值（必要时调用 stock-deep-dive）
2. 验证当前价位 vs 最新买入价（更新本库距离百分比）
3. 检查板块仓位上限是否还有空间
4. 决定 promote 到 focus 或继续观察
5. 更新本库标记（升级 / 维持 / 降级 / 剔除）

### 季度大维护
1. 全部已建档标的（约 35 只）扫一遍最新季报
2. 重新分档（A/B/C 是否变化）
3. 剔除已不符合投资逻辑的标的
4. 添加新建档但未入 focus 的标的

## 反共识铁律（避免常见误区）

1. **"我觉得 X 不错"不是触发条件** — 必须有客观宏观信号 + 估值条件双重确认
2. **"备选 ≠ 该买"** — 备选库是"等触发"，不是"现在加"。当前价位通常**不应该**贴近买入价
3. **板块多元化优先于单点机会** — 已超仓的板块即使出现"好机会"也应该克制
4. **降级比升级更重要** — focus list 应该是动态的，不是只增不减
5. **建档质量优于备选数量** — 35 只已建档够用了，新标的需要先做 deep-dive 再入备选
